## Описание интеграции

В рамках лабораторной работы в Telegram-бот `team_helper_bot` добавлена интеграция с внешним табличным источником данных: CSV-файлом сотрудников.  
Новый пользовательский сценарий:

1. Пользователь отправляет команду `/employees`.
2. Бот запрашивает имя сотрудника или отдел.
3. Бот выполняет поиск по CSV и возвращает найденные записи в удобном формате.

### Какой источник данных выбрали

Выбран файл `employees.csv` в корне проекта.

### Почему именно этот

- CSV прост в создании и редактировании без отдельной БД.
- Формат хорошо подходит для учебного кейса и небольших справочников.
- Удобно читать и фильтровать через `pandas`.
- Минимальные требования к инфраструктуре (достаточно одного файла).

### Структура данных

Используется структура:

```csv
name,department,role,email
Иван Иванов,HR,Manager,ivan@company.com
Борис Борисов,ML,Engineer,boris@company.com
```

Поля:

- `name` — имя сотрудника
- `department` — отдел
- `role` — должность
- `email` — корпоративная почта

### Как работает интеграция

1. В `bot.py` добавлено состояние диалога `EMPLOYEE_SEARCH_STATE`.
2. Добавлен обработчик входа `employees_entry()` для команды `/employees`.
3. Добавлен обработчик `employees_search()`, который:
   - читает CSV через `load_employees_data()`,
   - проверяет наличие обязательных колонок,
   - выполняет поиск по `name` и `department` без учета регистра,
   - формирует ответ по каждому найденному сотруднику.
4. Диалог подключен через `ConversationHandler` и зарегистрирован в `main()`.
5. Команда `/employees` добавлена в текст `/start`.

### Ключевые фрагменты кода

- Загрузка и валидация CSV:
  - `load_employees_data()`
- Старт диалога поиска:
  - `employees_entry()`
- Поиск и ответ пользователю:
  - `employees_search()`
- Подключение в роутинг бота:
  - `employees_conversation` + `app.add_handler(employees_conversation)`

### Используемые библиотеки

- `python-telegram-bot` — Telegram API и диалоги (`ConversationHandler`)
- `pandas` — чтение и фильтрация CSV
- `python-dotenv` — загрузка токена из `.env`
- стандартные модули Python: `os`, `json`, `logging`
